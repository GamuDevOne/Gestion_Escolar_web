Tarea: Página de cambio de contraseña obligatorio
Debes crear una página cambiar_password.html en la carpeta frontend/ que se muestre automáticamente cuando un usuario inicia sesión por primera vez
con su contraseña inicial. 
Esta página debe ser obligatoria — el usuario no puede saltársela ni acceder a su vista normal sin completarla.

Lo que debe tener la página:
Un formulario con tres campos:

Contraseña actual
Contraseña nueva
Confirmar contraseña nueva

Y un botón de guardar. El diseño debe seguir el estilo visual de login.css / index.html ya que es una página independiente del dashboard.

Archivos que tienes disponibles para trabajar:

frontend/css/login.css — usa este CSS como base para el diseño de la página
frontend/js/api.js — ya tiene API_URL y apiFetch listos para usar, inclúyelo antes de tu JS
frontend/js/login.js — referencia de cómo se hace un fetch al backend con manejo de errores


Cómo debe funcionar el JS de la página:

1. Al cargar la página, leer el localStorage para obtener el token y el rol del usuario
   - Si no hay sesión activa, redirigir a index.html
   - Si password_cambiada es true, redirigir directo a su vista (no necesita cambiar)

2. Al enviar el formulario:
   - Validar que los campos no estén vacíos
   - Validar que "contraseña nueva" y "confirmar contraseña" coincidan
   - Validar que la nueva contraseña tenga al menos 6 caracteres
   - Hacer un POST a: /auth/cambiar_password.php
     con el body: { password_actual, password_nueva }
     y el header Authorization: Bearer <token> (apiFetch lo hace automáticamente)

3. Si el servidor responde success: true
   - Actualizar en localStorage: password_cambiada = true
   - Mostrar mensaje de éxito (puede usar SweetAlert2 que ya está disponible)
   - Redirigir según el rol guardado en localStorage:
     admin     → admin.html
     profesor  → profesor.html
     estudiante → estudiante.html

4. Si el servidor responde error
   - Mostrar el mensaje de error que devuelve el servidor en json.error

Estructura de scripts al final del HTML:
<script src="https://cdn.jsdelivr.net/npm/sweetalert2@11"></script>
<script src="js/api.js"></script>
<script src="js/cambiar_password.js"></script>

Lo que devuelve el backend en cada caso:

Éxito:
json{ "success": true, "data": { "message": "Contraseña actualizada correctamente" } }

Error contraseña actual incorrecta:
json{ "success": false, "error": "La contraseña actual es incorrecta" }

Error contraseña igual a la anterior:
json{ "success": false, "error": "La nueva contraseña no puede ser igual a la actual" }

Lo que NO debes tocar — ya está hecho por el backend:

La lógica de validación de contraseñas está en el servidor
El token se maneja automáticamente con apiFetch
La columna password_cambiada en la base de datos ya existe
La redirección automática al login si el token expira ya la maneja api.js
